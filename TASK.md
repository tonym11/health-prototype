Goal:

Create a clickable mobile prototype that combines a Healtix-like health tracking structure with our +Life design system.



What we are testing:

Whether the interface logic of an existing health app and our visual/design language can work well together.



Core product logic:

\- Healtix-like layer = health data collection

\- +Life layer = AI health intelligence



The prototype should include 5 mobile-first screens:

\- Home

\- Labs

\- Biomarker Detail

\- AI Chat

\- Profile



Important constraints:

\- use mock data only

\- no backend

\- no auth

\- no external APIs

\- focus on interface, layout and navigation

\- keep components reusable

\- optimize for quick visual iteration



Design direction:

\- use our +Life design language

\- calm, scientific, minimal

\- avoid generic modern mobile UI

\- the result should feel like one unified product, not two stitched apps
